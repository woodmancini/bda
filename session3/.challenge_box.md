# Session 3 challenge box

## Data Analytics Challenges

### 1. Streaming preview for a huge file

You receive a 5 GB text file and want to preview the first 20 non-empty lines before deciding what to do next.

- Design the simplest generator for this.
- Explain why `readlines()` is a bad default here.
- What are the time and extra space complexities if the first 20 useful lines appear near the start?

### 2. First match with early stopping

You need the first line in `les_miserables.txt` that contains `"Fantine"`, but the file may later be replaced by a much larger corpus.

- Explain the role of `break`.
- Give best-case and worst-case time complexity.
- What would change if you needed **all** matching lines instead of the first one?

### 3. One-pass statistics

You need count, total characters, average line length, and longest line length.

- Can all of these be computed in one streaming pass?
- Which variables would you keep?
- What is the extra space complexity?

### 4. Repeated random access

You need line 10,000 once during setup, but then you need to use it 500 times during later analysis.

- Compare streaming from the start each time with loading lines once.
- Which approach has better total time after many repeated lookups?
- What space trade-off do you accept?

### 5. Tiny RAG retrieval design

You want to ask Gemini: "Who is Bishop Myriel?" using only retrieved context from the book.

- What should the retrieval step do before generation?
- Why might sending too few lines be a problem?
- Why might sending too many lines be a problem?
- How can `yield` help build context without loading the whole book?

## Complexity Challenges

### 1. One pass versus repeated passes

A script counts lines containing `"Jean"`, then counts lines containing `"Fantine"`, then counts lines containing `"Myriel"`.

- What is the complexity if you make three separate passes?
- How could you do it in one pass?
- What extra variables or data structures would you keep?

### 2. Duplicate detection

You want to detect duplicate lines in the first 10,000 lines of a file.

- What is the complexity of comparing every pair of lines?
- How could a `set` improve the time complexity?
- What space trade-off does the `set` introduce?

### 3. Subset explosion

An algorithm tries every possible subset of `n` candidate keywords to find the best search query.

- What complexity class best describes this?
- How many subsets are there for 10 keywords? For 30?
- Why does this become impractical quickly?

### 4. Permutation explosion

A route planner tries every possible order of `n` cities, or a scheduler tries every possible order of `n` jobs.

- What complexity class best describes this?
- Why is this worse than O(n²)?
- What kind of heuristic or shortcut might be acceptable in practice?

### 5. From naive recursion to dynamic programming

Naive recursive Fibonacci repeats the same calculations many times.

- Identify the overlapping subproblems.
- Explain how memoization changes the amount of repeated work.
- What is the trade-off between time and memory?
